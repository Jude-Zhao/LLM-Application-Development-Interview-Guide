# 链表

## 1. 核心思想

链表题的核心难点通常不在“思路有多复杂”，而在：

- 指针操作是否清楚
- 边界处理是否严谨

因为链表不像数组那样可以随机访问，所以常见技巧集中在：

- 虚拟头节点 `dummy`
- 快慢指针
- 反转链表
- 合并两个链表

很多链表题本质上都能拆成这几种基础操作。

---

## 2. 基础模板

### 2.1 虚拟头节点模板

```python
def solve(head):
    dummy = ListNode(0)
    dummy.next = head
    cur = head

    while cur:
        根据题意修改指针
        cur = cur.next

    return dummy.next
```

适用场景：

- 删除节点
- 插入节点
- 头节点可能发生变化

### 2.2 反转链表模板

```python
def reverseList(head):
    prev = None
    cur = head

    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt

    return prev
```

### 2.3 快慢指针模板

```python
def fast_slow(head):
    slow = head
    fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    return slow
```

适用场景：

- 找中点
- 判断是否有环
- 找倒数第 `k` 个节点

### 2.4 合并两个有序链表模板

```python
def mergeTwoLists(list1, list2):
    dummy = ListNode(0)
    tail = dummy

    while list1 and list2:
        if list1.val < list2.val:
            tail.next = list1
            list1 = list1.next
        else:
            tail.next = list2
            list2 = list2.next
        tail = tail.next

    tail.next = list1 if list1 else list2
    return dummy.next
```

## 3. 写链表题时最容易错的点

### 3.1 修改指针前没先保存下一跳

例如反转链表时，如果不先保存：

```python
nxt = cur.next
```

后面一改指针，就可能把剩余链表丢掉。

### 3.2 忘记头节点可能被删除或改变

这时最好直接使用 `dummy` 节点，统一处理。

### 3.3 快慢指针边界写错

常见循环条件是：

```python
while fast and fast.next:
```

少写一个判断就容易空指针。

### 3.4 画不清指针移动过程

链表题很适合边写边画：

- 当前节点是谁
- 下一跳是谁
- 修改后谁连向谁

不画很容易绕乱。

## 5. Hot 100 经典例题

## 5.1 206. 反转链表

题意：反转单链表。

### 思路

维护三个指针：

- `prev`：已经反转好的前半部分头节点
- `cur`：当前正在处理的节点
- `nxt`：提前保存的下一个节点

每次把 `cur.next` 指向 `prev`，然后整体向前推进。

### 代码

```python
def reverseList(head):
    prev = None
    cur = head

    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt

    return prev
```

### 这题训练的能力

- 掌握链表最基础的指针修改
- 为很多复杂链表题打基础

---

## 5.2 141. 环形链表

题意：判断链表是否有环。

### 思路

使用快慢指针：

- 慢指针每次走一步
- 快指针每次走两步

如果有环，两者最终一定会相遇。

### 代码

```python
def hasCycle(head):
    slow = head
    fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True

    return False
```

### 这题训练的能力

- 掌握快慢指针模型
- 理解“追及问题”在链表中的应用

## 5.3 21. 合并两个有序链表

题意：合并两个升序链表，并返回升序结果。

### 思路

使用虚拟头节点 `dummy`，每次比较两个链表当前节点：

- 谁更小，就接谁
- 然后对应指针前进

最后把剩余链表整体接上。

### 代码

```python
def mergeTwoLists(list1, list2):
    dummy = ListNode(0)
    tail = dummy

    while list1 and list2:
        if list1.val < list2.val:
            tail.next = list1
            list1 = list1.next
        else:
            tail.next = list2
            list2 = list2.next
        tail = tail.next

    tail.next = list1 if list1 else list2
    return dummy.next
```

### 这题训练的能力

- 熟悉 `dummy` 节点的使用
- 掌握链表合并的基本操作

## 5.4 19. 删除链表的倒数第 N 个结点

题意：删除链表的倒数第 `n` 个节点。

### 思路

使用双指针：

- `fast` 先走 `n` 步
- 然后 `fast` 和 `slow` 一起走
- 当 `fast` 到末尾时，`slow` 的下一个节点就是要删除的节点

为了统一删除头节点的情况，使用 `dummy`。

### 代码

```python
def removeNthFromEnd(head, n):
    dummy = ListNode(0)
    dummy.next = head
    fast = dummy
    slow = dummy

    for _ in range(n):
        fast = fast.next

    while fast.next:
        fast = fast.next
        slow = slow.next

    slow.next = slow.next.next
    return dummy.next
```

### 这题训练的能力

- 掌握“找倒数第 `k` 个节点”的双指针技巧
- 学会用 `dummy` 简化删除操作

## 5.5 160. 相交链表

题意：求两个链表相交的起始节点。

### 思路

让两个指针分别从两条链表头出发：

- 走到末尾后就跳到另一条链表头

这样两者总共走过的长度相同，如果有交点，最终会在交点相遇；如果没有交点，最终都会变成 `None`。

### 代码

```python
def getIntersectionNode(headA, headB):
    a = headA
    b = headB

    while a != b:
        a = a.next if a else headB
        b = b.next if b else headA

    return a
```

### 这题训练的能力

- 理解“长度对齐”的巧妙双指针方法
- 训练对链表路径长度的抽象能力

## 5.6 24. 两两交换链表中的节点

题意：每两个相邻节点交换一次。

### 思路

每次处理两个节点：

- `first`
- `second`

把它们的连接关系改成：

- `prev -> second -> first -> next_pair`

然后继续处理下一组。

### 代码

```python
def swapPairs(head):
    dummy = ListNode(0)
    dummy.next = head
    prev = dummy

    while prev.next and prev.next.next:
        first = prev.next
        second = prev.next.next

        prev.next = second
        first.next = second.next
        second.next = first

        prev = first

    return dummy.next
```

### 这题训练的能力

- 练习局部链表重连
- 学会在修改前先看清每个指针指向

## 7. 一句话总结链表

做链表题时，优先问自己四个问题：

- 要不要先加一个 `dummy` 节点？
- 当前题更适合双指针、快慢指针，还是反转链表？
- 修改指针前，下一跳有没有先保存？
- 头节点变化、空链表、单节点这些边界有没有处理？

这四个问题想清楚，链表题就不会那么容易绕乱。
