# 哈希与字符串

## 1. 核心思想

哈希和字符串经常放在一起，是因为很多字符串题的高效做法都依赖哈希结构。

- 哈希表解决的是“快速查找、统计、去重、映射”
- 字符串题解决的是“字符处理、模式匹配、子串判断、编码解码”

在面试里，这两类题经常组合出现：

- 统计字符出现次数
- 判断两个字符串是否满足某种映射关系
- 找子串、分组、去重
- 用集合判断是否访问过

这类题最核心的思路通常是：

> 用哈希结构，把原本需要反复扫描的操作降到 `O(1)`。

## 2. 基础模板

### 2.1 计数哈希模板

```python
from collections import Counter


def count_chars(s):
    cnt = Counter(s)
    return cnt
```

适用场景：

- 词频统计
- 异位词判断
- 出现次数比较

### 2.2 位置映射模板

```python
def first_index(nums):
    pos = {}

    for i, x in enumerate(nums):
        if x not in pos:
            pos[x] = i

    return pos
```

适用场景：

- 两数之和
- 记录第一次出现位置
- 前缀和 + 哈希

### 2.3 字符串遍历模板

```python
def process(s):
    for i, ch in enumerate(s):
        根据字符做处理
```

### 2.4 分组哈希模板

```python
from collections import defaultdict


def group(words):
    groups = defaultdict(list)

    for word in words:
        key = tuple(sorted(word))
        groups[key].append(word)

    return list(groups.values())
```

适用场景：

- 字母异位词分组
- 分类统计

## 3. 写哈希和字符串题时最容易错的点

### 3.1 键的设计不合理

哈希题很多时候不是不会用字典，而是不会设计 key。

例如异位词分组时，常见 key 有：

- 排序后的字符串
- 26 个字母频次数组转成元组

### 3.2 只想到暴力匹配，没想到先做统计

字符串题常常可以先把信息压缩成：

- 频次
- 下标
- 前缀状态

这样就不需要重复扫描。

### 3.3 忘记处理重复元素

例如：

- 两数之和里同一个元素不能重复使用
- 最长连续序列里要去重
- 字母异位词分组要把多个结果放进同一个桶

### 3.4 字符串不可变带来的开销

在 Python 中，字符串是不可变的。

如果频繁拼接大字符串，可能带来额外开销。很多题更适合：

- 用列表收集
- 最后 `''.join(...)`

## 5. Hot 100 经典例题

## 5.1 1. 两数之和

题意：在数组中找出两个数，使它们的和等于目标值。

### 思路

一边遍历，一边把已经见过的数放进哈希表。

对当前数字 `x`：

- 需要找的配对值是 `target - x`
- 如果这个值之前出现过，直接返回答案
- 否则把当前值和下标存入哈希表

### 代码

```python
def twoSum(nums, target):
    pos = {}

    for i, x in enumerate(nums):
        need = target - x
        if need in pos:
            return [pos[need], i]
        pos[x] = i
```

### 这题训练的能力

- 用空间换时间
- 理解“边遍历边查询”的哈希写法

## 5.2 49. 字母异位词分组

题意：把互为字母异位词的字符串分到同一组。

### 思路

异位词的特征是：

- 排序后字符串相同

所以可以把排序结果作为哈希表的 key。

### 代码

```python
from collections import defaultdict


def groupAnagrams(strs):
    groups = defaultdict(list)

    for s in strs:
        key = ''.join(sorted(s))
        groups[key].append(s)

    return list(groups.values())
```

### 这题训练的能力

- 设计哈希 key
- 做分类聚合

## 5.3 128. 最长连续序列

题意：给定一个未排序数组，求最长连续序列长度。

### 思路

先把所有数放进集合，之后只从“序列起点”开始往后扩展。

一个数 `x` 是序列起点，当且仅当：

- `x - 1` 不在集合里

这样每个连续序列只会被遍历一次。

### 代码

```python
def longestConsecutive(nums):
    s = set(nums)
    ans = 0

    for x in s:
        if x - 1 not in s:
            cur = x
            length = 1

            while cur + 1 in s:
                cur += 1
                length += 1

            ans = max(ans, length)

    return ans
```

### 这题训练的能力

- 用集合快速判断存在性
- 避免重复枚举

## 5.4 5. 最长回文子串

题意：返回字符串中的最长回文子串。

### 思路

这题虽然不靠哈希，但属于高频字符串题。

最常见的面试写法是中心扩展：

- 以一个字符为中心，向两边扩
- 以两个字符之间为中心，向两边扩

因为回文可能是奇数长度，也可能是偶数长度。

### 代码

```python
def longestPalindrome(s):
    def expand(left, right):
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1
            right += 1
        return s[left + 1:right]

    ans = ""

    for i in range(len(s)):
        s1 = expand(i, i)
        s2 = expand(i, i + 1)
        ans = max(ans, s1, s2, key=len)

    return ans
```

### 这题训练的能力

- 掌握字符串区间扩展
- 识别奇偶长度回文

## 5.5 76. 最小覆盖子串

题意：找到 `s` 中包含 `t` 所有字符的最短子串。

### 思路

这题是“哈希 + 滑动窗口”的代表题。

窗口内维护两个信息：

- `need`：目标字符频次
- `window`：当前窗口字符频次

当窗口已经覆盖所有要求时：

- 尝试收缩左边界
- 更新最短答案

### 代码

```python
from collections import Counter, defaultdict


def minWindow(s, t):
    need = Counter(t)
    window = defaultdict(int)
    required = len(need)
    formed = 0
    left = 0
    start = 0
    min_len = float("inf")

    for right, ch in enumerate(s):
        window[ch] += 1
        if ch in need and window[ch] == need[ch]:
            formed += 1

        while formed == required:
            if right - left + 1 < min_len:
                min_len = right - left + 1
                start = left

            left_ch = s[left]
            window[left_ch] -= 1
            if left_ch in need and window[left_ch] < need[left_ch]:
                formed -= 1
            left += 1

    return "" if min_len == float("inf") else s[start:start + min_len]
```

### 这题训练的能力

- 把哈希统计嵌入滑动窗口
- 处理“窗口何时满足条件”
