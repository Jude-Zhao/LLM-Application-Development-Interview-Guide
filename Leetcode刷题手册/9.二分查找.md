# 二分查找



## 1. 核心思想



二分查找的前提通常是：



- 数组或答案空间具有单调性

- 每次可以根据 `mid` 的判断结果，把搜索区间缩小一半



它不只用于“在有序数组里找某个数”，还常用于：



- 查找某个目标值是否存在

- 查找第一个满足条件的位置

- 查找最后一个满足条件的位置

- 在答案空间上二分答案



时间复杂度通常为 `O(log n)`。



---



## 2. 基础模板



### 2.1 查找某个目标值



```python

def binary_search(nums, target):

   left, right = 0, len(nums) - 1



   while left <= right:

       mid = left + (right - left) // 2



       if nums[mid] == target:

           return mid

       elif nums[mid] < target:

           left = mid + 1

       else:

           right = mid - 1



   return -1

```



适用场景：



- 数组整体有序

- 只需要判断是否存在或返回任意一个位置



### 2.2 查找左边界



```python

def left_bound(nums, target):

   left, right = 0, len(nums) - 1

   ans = -1



   while left <= right:

       mid = left + (right - left) // 2

       if nums[mid] >= target:

           if nums[mid] == target:

               ans = mid

           right = mid - 1

       else:

           left = mid + 1



   return ans

```



### 2.3 查找右边界



```python

def right_bound(nums, target):

   left, right = 0, len(nums) - 1

   ans = -1



   while left <= right:

       mid = left + (right - left) // 2

       if nums[mid] <= target:

           if nums[mid] == target:

               ans = mid

           left = mid + 1

       else:

           right = mid - 1



   return ans

```

## 3. 写二分时最容易错的点



### 3.1 区间到底是闭区间还是左闭右开



最常见的是闭区间写法：



- 初始化：`left = 0, right = len(nums) - 1`

- 循环条件：`left <= right`

- 收缩区间时：

 - `left = mid + 1`

 - `right = mid - 1`



如果你改成左闭右开区间，就要整套一起改，不能混用。



### 3.2 `mid` 的计算



推荐统一写成：



```python

mid = left + (right - left) // 2

```



这样逻辑稳定，也避免某些语言里整数溢出的问题。



### 3.3 看到“有序但不完全有序”也要想到二分



例如：



- 旋转排序数组

- 某一段满足条件，另一段不满足条件

- 答案空间具有单调性



这类题目不一定是标准有序数组，但依然可以二分。

## 4. Hot 100 经典例题



## 4.1 35. 搜索插入位置



题意：在有序数组中找到目标值；若不存在，返回它应该插入的位置。



### 思路



本质上是在找“第一个大于等于 `target` 的位置”。



- 如果找到目标值，返回其下标

- 如果没找到，最终 `left` 停留的位置就是插入点



### 代码



```python

def searchInsert(nums, target):

   left, right = 0, len(nums) - 1



   while left <= right:

       mid = left + (right - left) // 2

       if nums[mid] < target:

           left = mid + 1

       else:

           right = mid - 1



   return left

```



### 为什么最后返回 `left`



循环结束时：



- `right` 指向最后一个小于 `target` 的位置

- `left` 指向第一个大于等于 `target` 的位置



因此插入位置就是 `left`。

## 4.2 34. 在排序数组中查找元素的第一个和最后一个位置



题意：给定一个升序数组，返回目标值的起始和结束位置。



### 思路



这题本质是：



- 一次二分找左边界

- 一次二分找右边界



### 代码



```python

def searchRange(nums, target):

   def left_bound():

       left, right = 0, len(nums) - 1

       ans = -1

       while left <= right:

           mid = left + (right - left) // 2

           if nums[mid] >= target:

               if nums[mid] == target:

                   ans = mid

               right = mid - 1

           else:

               left = mid + 1

       return ans



   def right_bound():

       left, right = 0, len(nums) - 1

       ans = -1

       while left <= right:

           mid = left + (right - left) // 2

           if nums[mid] <= target:

               if nums[mid] == target:

                   ans = mid

               left = mid + 1

           else:

               right = mid - 1

       return ans



   return [left_bound(), right_bound()]

```



### 这题训练的能力



- 同一个问题拆成两个边界问题

- 明确“满足条件时继续向左找”还是“继续向右找”

## 4.3 33. 搜索旋转排序数组



题意：升序数组在某个点旋转后，查找目标值下标。



例如：`[4,5,6,7,0,1,2]`



### 关键观察



虽然整体不是完全有序，但每次二分时，`mid` 一定会落在某个有序区间里：



- 如果 `nums[left] <= nums[mid]`，说明左半段有序

- 否则右半段有序



再判断目标值是否落在有序区间中，决定往哪边继续搜。



### 代码



```python

def search(nums, target):

   left, right = 0, len(nums) - 1



   while left <= right:

       mid = left + (right - left) // 2



       if nums[mid] == target:

           return mid



       if nums[left] <= nums[mid]:

           if nums[left] <= target < nums[mid]:

               right = mid - 1

           else:

               left = mid + 1

       else:

           if nums[mid] < target <= nums[right]:

               left = mid + 1

           else:

               right = mid - 1



   return -1

```



### 易错点



- 不要只想着“整体不有序就不能二分”

- 关键是每轮能不能找到一半有序区间

## 4.4 153. 寻找旋转排序数组中的最小值



题意：升序数组旋转后，找最小值。



### 思路



比较 `nums[mid]` 和 `nums[right]`：



- 如果 `nums[mid] > nums[right]`，最小值在右边

- 否则最小值在左边，包括 `mid`



### 代码



```python

def findMin(nums):

   left, right = 0, len(nums) - 1



   while left < right:

       mid = left + (right - left) // 2

       if nums[mid] > nums[right]:

           left = mid + 1

       else:

           right = mid



   return nums[left]

```



### 为什么这里用 `while left < right`



因为最后要收缩到唯一答案位置，区间缩成一个点即可结束。

## 5. 如何判断一道题能不能用二分



看到下面信号时，优先考虑二分：



- 明确给出“有序数组”

- 题目要求 `O(log n)` 时间复杂度

- 某个性质随着下标变化具有单调性

- 某个答案“越大越满足”或“越小越满足”



可以先问自己两个问题：



1. 搜索空间是什么？

2. 判断结果是否具有单调性？



只要这两个问题能回答清楚，通常就能设计出二分。
