# 堆

## 1. 核心思想

堆（Heap）本质上是一种：

- 可以快速取出最值的数据结构
- 常用于动态维护“当前最大”或“当前最小”

在 Python 里，`heapq` 默认是**小根堆**，也就是：

- 堆顶永远是最小元素

如果想模拟大根堆，通常做法是：

- 往堆里放相反数

堆适合的典型场景：

- 动态维护前 `k` 大 / 前 `k` 小
- 不断取出当前最值
- 多路归并
- 数据流中维护中位数或第 `k` 大值

---

## 2. 基础知识

### 2.1 常用操作

```python
import heapq

heap = []

heapq.heappush(heap, x)      # 入堆
top = heapq.heappop(heap)    # 弹出堆顶最小值
peek = heap[0]               # 查看堆顶
```

### 2.2 维护固定大小的堆

```python
import heapq


def top_k(nums, k):
    heap = []

    for num in nums:
        heapq.heappush(heap, num)
        if len(heap) > k:
            heapq.heappop(heap)

    return heap[0]
```

这个模板的含义是：

- 始终维护一个大小为 `k` 的小根堆
- 堆里保存当前最大的 `k` 个数
- 堆顶就是这 `k` 个数里最小的那个，也就是第 `k` 大

### 2.3 多路归并模板

```python
import heapq


def merge(lists):
    heap = []

    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    while heap:
        _, i, node = heapq.heappop(heap)
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
```

适用场景：

- 合并多个有序链表
- 合并多个有序数组

## 3. 什么时候想到堆

### 3.1 题目要求反复取最大 / 最小

如果每次都排序，复杂度通常太高。

堆可以把：

- 取最值降到 `O(1)`
- 插入和删除降到 `O(log n)`

### 3.2 题目要求维护前 `k` 个元素

这是堆最典型的应用之一。

常见问法：

- 第 `k` 大元素
- 出现频率前 `k` 高
- 最接近原点的前 `k` 个点

### 3.3 题目有多个“有序来源”

例如：

- 多个升序链表
- 多个升序数组

这时通常考虑堆做多路归并。

---

## 4. 写堆题时最容易错的点

### 4.1 忘记 Python 默认是小根堆

想要大根堆时，要么：

- 存负数
- 存自定义比较项

### 4.2 堆里元组的比较规则没想清楚

Python 比较元组时，会按顺序逐项比较。

所以常见写法是：

- `(value, index)`
- `(freq, num)`
- `(node.val, list_id, node)`

其中加 `index` 或 `list_id`，往往是为了避免后面的对象不能直接比较。

### 4.3 堆和排序分不清

如果题目只是：

- 一次性找最大 / 最小
- 全部元素排序后直接输出

那不一定需要堆。

堆更适合：

- 动态过程
- 边处理边维护最值

## 5. Hot 100 经典例题

## 5.1 215. 数组中的第 K 个最大元素

题意：求数组中的第 `k` 个最大元素。

### 思路

维护一个大小为 `k` 的小根堆：

- 如果堆元素不足 `k` 个，直接加入
- 如果堆满了，且当前元素比堆顶大，就替换堆顶

最后堆顶就是第 `k` 大元素。

### 代码

```python
import heapq


def findKthLargest(nums, k):
    heap = []

    for num in nums:
        heapq.heappush(heap, num)
        if len(heap) > k:
            heapq.heappop(heap)

    return heap[0]
```

### 这题训练的能力

- 理解“固定大小堆”的经典模型
- 掌握前 `k` 类问题的标准写法

## 5.2 347. 前 K 个高频元素

题意：返回数组中出现频率前 `k` 高的元素。

### 思路

先统计每个数字的出现次数，再维护一个大小为 `k` 的小根堆：

- 堆中按照频率排序
- 堆顶始终是当前前 `k` 高频元素里频率最小的那个

### 代码

```python
import heapq
from collections import Counter


def topKFrequent(nums, k):
    counter = Counter(nums)
    heap = []

    for num, freq in counter.items():
        heapq.heappush(heap, (freq, num))
        if len(heap) > k:
            heapq.heappop(heap)

    return [num for _, num in heap]
```

### 这题训练的能力

- 把“频率统计”和“固定大小堆”结合起来
- 认识堆中存元组的常见方式

## 5.3 23. 合并 K 个升序链表

题意：给定多个升序链表，合并成一个升序链表。

### 思路

这题是典型的多路归并：

- 把每个链表的当前头结点放入堆中
- 每次弹出最小节点接到答案后面
- 再把该节点的下一个节点入堆

### 代码

```python
import heapq


def mergeKLists(lists):
    dummy = ListNode(0)
    tail = dummy
    heap = []

    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    while heap:
        _, i, node = heapq.heappop(heap)
        tail.next = node
        tail = tail.next

        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next
```

### 这题训练的能力

- 掌握“多个有序来源”的堆写法
- 理解为什么堆里常常要额外带一个下标

## 5.4 295. 数据流的中位数

题意：不断加入数字，要求能快速返回中位数。

### 思路

维护两个堆：

- 左边一个大根堆，保存较小的一半
- 右边一个小根堆，保存较大的一半

并保证：

- 两边元素个数差不超过 `1`
- 左边所有元素都不大于右边所有元素

### 代码

```python
import heapq


class MedianFinder:
    def __init__(self):
        self.left = []
        self.right = []

    def addNum(self, num):
        heapq.heappush(self.left, -num)
        heapq.heappush(self.right, -heapq.heappop(self.left))

        if len(self.right) > len(self.left):
            heapq.heappush(self.left, -heapq.heappop(self.right))

    def findMedian(self):
        if len(self.left) > len(self.right):
            return -self.left[0]
        return (-self.left[0] + self.right[0]) / 2
```

### 这题训练的能力

- 掌握双堆模型
- 理解“左半边最大值”和“右半边最小值”的平衡维护

## 7. 一句话总结堆

做堆题时，优先问自己三个问题：

- 题目是不是在反复取当前最值？
- 是不是要动态维护前 `k` 个元素？
- 是不是有多个有序序列需要同时归并？

如果答案是“是”，堆往往就是首选。
