# 回溯



## 1. 核心思想



回溯本质上是一种“试错 + 撤销选择”的搜索方法，通常用于：



- 枚举所有可能方案

- 在所有路径中找到满足条件的解

- 组合、排列、子集、切割问题



它的基本过程是：



1. 先做一个选择

2. 递归进入下一层

3. 递归返回后撤销这个选择

4. 再尝试别的选择



所以回溯常被概括成一句话：



> 路径 + 选择列表 + 结束条件



---



## 2. 通用模板



```python

def backtrack(路径, 选择列表):

   if 满足结束条件:

       收集答案

       return



   for 选择 in 选择列表:

       if 当前选择不合法:

           continue



       做选择

       backtrack(新路径, 新选择列表)

       撤销选择

```



写题时最重要的是想清楚三件事：



- 当前这层递归表示什么

- 这一层有哪些选择

- 递归返回时要撤销什么状态



---



## 3. 回溯常见类别



### 3.1 子集问题



特点：



- 每个元素只有“选 / 不选”两种状态

- 不强调顺序



典型题：`78. 子集`



### 3.2 组合问题



特点：



- 从若干元素中选出若干个

- 不强调顺序

- 往往通过 `startIndex` 控制不能回头选



典型题：`39. 组合总和`



### 3.3 排列问题



特点：



- 顺序重要

- 每个位置都要从“还没用过的元素”中选



典型题：`46. 全排列`



### 3.4 切割问题



特点：



- 把一个字符串或数组切成若干段

- 每一段要满足某种条件



典型题：`131. 分割回文串`



---



## 4. 回溯的两个关键技巧



## 4.1 路径 `path`



用于记录当前已经做出的选择。



例如在组合题中：



- `path = [2, 3]` 表示当前已经选了 2 和 3



递归过程中常见写法：



```python

path.append(x)

backtrack(...)

path.pop()

```



## 4.2 使用状态数组或起始下标



根据题目类型不同，通常有两种控制方式：



- `used[]`：常用于排列，表示某个元素是否已被使用

- `startIndex`：常用于组合/子集，避免重复选择前面的元素



---



## 5. Hot 100 经典例题



## 5.1 46. 全排列



题意：给定不含重复数字的数组，返回所有可能的全排列。



### 思路



排列问题的关键是：



- 顺序不同算不同答案

- 每一层都要从“还没使用过的元素”中继续选



因此我们需要：



- 一个 `path` 表示当前排列

- 一个 `used` 数组表示元素是否已选过



### 代码



```python

def permute(nums):

   res = []

   path = []

   used = [False] * len(nums)



   def backtrack():

       if len(path) == len(nums):

           res.append(path[:])

           return



       for i in range(len(nums)):

           if used[i]:

               continue



           used[i] = True

           path.append(nums[i])



           backtrack()



           path.pop()

           used[i] = False



   backtrack()

   return res

```



### 这题训练的能力



- 理解“每层都重新从头枚举”

- 熟悉 `used[]` 的控制方式

## 5.2 78. 子集



题意：给你一个数组，返回所有子集。



### 思路



子集问题的关键是：



- 每个节点本身就是一个合法答案

- 不需要等到走到叶子节点才收集结果



所以每次进入递归时，都可以先把当前 `path` 放进答案。



### 代码



```python

def subsets(nums):

   res = []

   path = []



   def backtrack(start):

       res.append(path[:])



       for i in range(start, len(nums)):

           path.append(nums[i])

           backtrack(i + 1)

           path.pop()



   backtrack(0)

   return res

```



### 为什么这里要用 `start`



因为子集不关心顺序，选过当前位置后，下一层只能从后面继续选，避免出现重复结果。

## 5.3 39. 组合总和



题意：给定无重复元素数组 `candidates` 和目标值 `target`，找出和为目标值的所有组合。一个数字可以重复使用。



### 思路



这题是“组合 + 剪枝”的典型题：



- 因为顺序不重要，所以用 `start` 控制搜索范围

- 因为元素可以重复使用，所以递归时仍然传 `i`

- 如果当前和已经大于目标值，可以直接停止这条分支



### 代码



```python

def combinationSum(candidates, target):

   res = []

   path = []

   candidates.sort()



   def backtrack(start, total):

       if total == target:

           res.append(path[:])

           return



       for i in range(start, len(candidates)):

           num = candidates[i]



           if total + num > target:

               break



           path.append(num)

           backtrack(i, total + num)

           path.pop()



   backtrack(0, 0)

   return res

```



### 这题里的剪枝



先排序，再在循环中判断：



```python

if total + num > target:

   break

```



因为后面的数只会更大，所以可以直接终止本层循环。

## 5.4 17. 电话号码的字母组合



题意：给定数字字符串，返回它能表示的所有字母组合。



### 思路



这是非常标准的“按层选择”回溯题：



- 第 1 层从第 1 个数字对应的字母中选一个

- 第 2 层从第 2 个数字对应的字母中选一个

- 当路径长度等于数字串长度时，得到一个答案



### 代码



```python

def letterCombinations(digits):

   if not digits:

       return []



   mapping = {

       "2": "abc", "3": "def", "4": "ghi", "5": "jkl",

       "6": "mno", "7": "pqrs", "8": "tuv", "9": "wxyz"

   }



   res = []

   path = []



   def backtrack(index):

       if index == len(digits):

           res.append("".join(path))

           return



       for ch in mapping[digits[index]]:

           path.append(ch)

           backtrack(index + 1)

           path.pop()



   backtrack(0)

   return res

```



### 这题训练的能力



- 理解“递归深度就是处理到第几个字符”

- 理解回溯不一定是数组选数，也可以是字符串按位决策

## 6. 回溯常见剪枝方式



回溯通常比二分更依赖剪枝，否则很容易超时。



常见剪枝方式：



- 当前路径已经不可能成为答案，直接返回

- 数组排序后，利用大小关系提前终止循环

- 遇到重复元素时跳过，防止重复答案

- 用 `used[]`、集合、哈希表记录状态，避免重复搜索



剪枝的本质是：



> 尽早停止不可能成功的分支



---



## 7. 写回溯题时的固定思考顺序



可以按下面顺序分析：



1. 递归函数的参数是什么？

2. 结束条件是什么？

3. 每一层从哪里开始枚举？

4. 做选择时修改了哪些状态？

5. 返回上一层时要撤销哪些状态？



如果这五个问题能答清楚，回溯代码一般就能写出来。



---



## 8. 易错点



### 8.1 忘记撤销选择



例如加了 `path.append(...)`，却忘了 `path.pop()`，会导致路径污染。



### 8.2 结果直接 `append(path)`



应该写成：



```python

res.append(path[:])

```



因为 `path` 是可变对象，直接加入会在后续递归中被修改。



### 8.3 组合题和排列题写混



要区分：



- 排列：通常用 `used[]`

- 组合 / 子集：通常用 `startIndex`



### 8.4 剪枝条件写错位置



例如应该在进入递归前剪枝，却写到了递归后，就起不到效果。
