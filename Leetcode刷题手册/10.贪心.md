# 贪心

## 1. 核心思想

贪心算法的核心是：

- 每一步都做当前看起来最优的选择
- 不回头修改之前的决策
- 希望通过一系列局部最优，得到全局最优

它和动态规划最大的区别在于：

- 动态规划会保留多种状态，最后比较出最优解
- 贪心只保留当前最优选择，直接往前走

所以贪心题最关键的问题不是“怎么写”，而是：

> 为什么当前这一步的局部最优，能够推出最终的全局最优？

常见适用场景：

- 只关心当前最优边界
- 题目具有明显的单调性
- 排序后可以逐步决策
- 每一步选择不会破坏后续最优性

---

## 2. 贪心的常见思考方式

### 2.1 从题目目标反推“每一步该贪什么”

例如：

- 想让跳跃次数最少，就尽量让每一步覆盖范围最大
- 想让区间数量最多，就优先选择结束最早的区间
- 想让利润最大，就尽量保留更低的成本

### 2.2 先排序，再贪心

很多贪心题不排序就很难做。

常见排序方式：

- 按左端点排序
- 按右端点排序
- 按长度排序
- 按权重或收益排序

### 2.3 维护一个“当前最优状态”

很多题不需要复杂数据结构，只要维护：

- 当前最远可达位置
- 当前最低价格
- 当前剩余油量
- 当前可合并区间的右边界

### 2.4 贪心常常配合“证明”

常见证明思路：

- 交换论证
- 局部最优不会影响后续最优
- 如果不这样选，答案不会更好

## 3. 基础模板

### 3.1 扫描数组型贪心

```python
def greedy(nums):
    ans = 初始值
    state = 初始状态

    for x in nums:
        根据当前状态做局部最优选择
        更新答案或状态

    return ans
```

适用场景：

- 一次遍历就能维护最优状态
- 当前决策只依赖前面信息

### 3.2 排序后贪心

```python
def greedy(intervals):
    intervals.sort(key=lambda x: 排序标准)

    ans = 0
    state = 初始状态

    for item in intervals:
        if 满足条件:
            选择它
            更新状态
            ans += 1

    return ans
```

适用场景：

- 区间问题
- 资源分配问题
- 合并 / 选择类问题

## 4. 写贪心时最容易错的点

### 4.1 只会猜策略，不会验证策略

贪心最怕“感觉这样对”，但没有依据。

写之前至少要想清楚：

- 为什么这样选不会吃亏
- 后面的选择会不会被当前选择破坏

### 4.2 该排序却没排序

很多区间类、选择类题，如果不先排序，局部最优根本没有意义。

### 4.3 把动态规划题误判成贪心题

如果当前局部最优会影响未来，并且不能证明最优性，那往往不是贪心。

### 4.4 状态更新顺序写错

例如跳跃游戏 II 中：

- 什么时候更新当前边界
- 什么时候增加步数

顺序写错就容易多算或少算一步。

## 5. Hot 100 经典例题

## 5.1 121. 买卖股票的最佳时机

题意：只能买卖一次，求最大利润。

### 思路

贪心地维护：

- 到当前位置为止的最低买入价格
- 当前位置卖出时的最大利润

也就是：

- 如果今天价格更低，就更新最低成本
- 如果今天卖出更划算，就更新答案

### 代码

```python
def maxProfit(prices):
    min_price = float("inf")
    ans = 0

    for price in prices:
        min_price = min(min_price, price)
        ans = max(ans, price - min_price)

    return ans
```

### 这题训练的能力

- 维护历史最优状态
- 理解“一边遍历一边更新答案”的贪心写法

## 5.2 55. 跳跃游戏

题意：每个位置表示最多能跳多远，判断能否到达最后一个位置。

### 思路

维护当前能到达的最远位置 `max_reach`。

遍历数组时：

- 如果当前位置 `i` 已经超过 `max_reach`，说明到不了这里
- 否则更新 `max_reach = max(max_reach, i + nums[i])`

只要始终能把最远边界往后推，就能到终点。

### 代码

```python
def canJump(nums):
    max_reach = 0

    for i, step in enumerate(nums):
        if i > max_reach:
            return False
        max_reach = max(max_reach, i + step)

    return True
```

### 这题训练的能力

- 维护区间覆盖范围
- 理解“只看当前最远能到哪”这一类贪心

## 5.3 45. 跳跃游戏 II

题意：保证可以到达终点，求最少跳跃次数。

### 思路

这题仍然维护“覆盖范围”，但需要两个边界：

- `cur_end`：当前这一步最远能覆盖到哪里
- `next_end`：再多看一层后，最远能扩展到哪里

遍历到 `cur_end` 时，说明当前这一步已经走完，必须再跳一次。

### 代码

```python
def jump(nums):
    steps = 0
    cur_end = 0
    next_end = 0

    for i in range(len(nums) - 1):
        next_end = max(next_end, i + nums[i])
        if i == cur_end:
            steps += 1
            cur_end = next_end

    return steps
```

### 这题训练的能力

- 理解按“层”推进的贪心
- 明白为什么走到当前边界时才加一步

## 5.4 763. 划分字母区间

题意：把字符串尽可能分成多个片段，同一个字母最多只出现在一个片段中。

### 思路

先记录每个字符最后出现的位置。

遍历字符串时维护当前片段右边界 `end`：

- 每看到一个字符，就把 `end` 更新为该字符最后出现位置的最大值
- 当遍历到 `i == end`，说明当前片段可以闭合

### 代码

```python
def partitionLabels(s):
    last = {ch: i for i, ch in enumerate(s)}
    res = []
    start = 0
    end = 0

    for i, ch in enumerate(s):
        end = max(end, last[ch])
        if i == end:
            res.append(end - start + 1)
            start = i + 1

    return res
```

### 这题训练的能力

- 维护当前片段必须覆盖到的最右边界
- 理解“边遍历边决定切割点”的贪心

## 5.5 53. 最大子数组和

题意：求连续子数组的最大和。

### 思路

这题也可以从贪心角度理解：

- 如果当前前缀和已经是负数，那它只会拖累后面
- 所以当累计和小于 `0` 时，直接从当前位置重新开始

它也可以写成动态规划，但用贪心理解会更直观。

### 代码

```python
def maxSubArray(nums):
    cur_sum = nums[0]
    ans = nums[0]

    for num in nums[1:]:
        cur_sum = max(num, cur_sum + num)
        ans = max(ans, cur_sum)

    return ans
```

### 这题训练的能力

- 理解“有害前缀直接丢弃”的局部最优思想
- 区分同一道题的贪心解和 DP 解

## 5.6 134. 加油站

题意：环形路线，每个站点可获得油量 `gas[i]`，去下一个站消耗 `cost[i]`，问从哪个起点出发能绕一圈。

### 思路

关键结论：

- 如果总油量小于总消耗，一定无解
- 如果从起点 `start` 出发，到 `i` 位置失败了，那么 `start ~ i` 中任何点都不可能作为合法起点

因此一旦当前油量变成负数，就直接把起点跳到下一位。

### 代码

```python
def canCompleteCircuit(gas, cost):
    if sum(gas) < sum(cost):
        return -1

    start = 0
    tank = 0

    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        if tank < 0:
            start = i + 1
            tank = 0

    return start
```

### 这题训练的能力

- 理解“失败区间整体排除”的贪心结论
- 学会利用题目中的整体约束

## 7. 一句话总结贪心

做贪心题时，优先问自己三个问题：

- 当前这一步到底该贪什么？
- 为什么这样贪不会影响最终最优解？
- 是否需要先排序或维护某个边界？

如果这三个问题能答清楚，贪心题就会顺很多。
