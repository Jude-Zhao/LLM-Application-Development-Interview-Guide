# LeetCode刷题核心精髓

## 一、二分查找

**核心步骤:**
1. 确定搜索空间和单调性
2. 定义左右边界(闭区间: left=0, right=len-1)
3. 循环条件: left <= right
4. 计算mid: mid = left + (right-left)//2
5. 根据mid判断结果收缩区间
6. 返回目标或边界

**关键点:**
- 区间定义要统一(闭区间或左闭右开)
- mid计算防止溢出
- 有序但不完全有序也可二分
- 左边界: 满足条件继续向左找
- 右边界: 满足条件继续向右找

---

## 二、回溯

**核心步骤:**
1. 定义递归函数(路径+选择列表+结束条件)
2. 满足结束条件时收集答案并返回
3. 遍历选择列表
4. 做选择→递归→撤销选择
5. 使用path记录路径,used[]或startIndex控制重复

**关键点:**
- 子集: 每个节点都是答案,用startIndex
- 组合: 用startIndex避免重复,元素可重复用i
- 排列: 用used[]标记已使用
- 切割: 判断子串是否满足条件
- 剪枝: 尽早停止不可能成功的分支

---

## 三、多维动态规划

**核心步骤:**
1. 定义dp数组(二维或更多维度)
2. 初始化边界条件
3. 确定状态转移方程
4. 按正确顺序遍历
5. 返回结果(考虑空间优化)

**关键点:**
- 路径类: dp[i][j] = min/max(dp[i-1][j], dp[i][j-1]) + cost
- 匹配类: dp[i][j] = dp[i-1][j-1] + match(i,j)
- 选择类: dp[i][j] = max(dp[i-1][j], dp[i][j-1], dp[i-1][j-1] + val)
- 空间优化: 滚动数组或一维数组
- 边界初始化要完整

---

## 四、二叉树

**核心步骤:**
1. 确定递归函数定义(返回值类型)
2. 处理空节点边界
3. 前中后序或层序遍历
4. 整合左右子树信息
5. 返回当前节点结果

**关键点:**
- 前序: 根→左→右
- 中序: 左→根→右(BST中序有序)
- 后序: 左→右→根(自底向上)
- 层序: 用队列BFS,记录每层节点数
- 信息回传: 左右子树返回信息,当前节点整合
- 修改类: 先处理当前节点,再递归

---

## 五、动态规划

**核心步骤:**
1. 定义dp数组含义
2. 找状态转移方程
3. 确定初始化
4. 确定遍历顺序
5. 确定返回值

**关键点:**
- 线性DP: dp[i]依赖前几个位置
- 背包DP: 0/1背包倒序遍历,完全背包正序遍历
- 前缀DP: dp[i]表示前i个元素是否可行/最优
- 二维路径: 从上方、左方转移
- 双序列DP: dp[i][j]表示两个前缀的关系
- 区间DP: 按区间长度递推

---

## 六、堆

**核心步骤:**
1. 确定需要动态维护最值
2. 维护固定大小堆(前k大/小)
3. 多路归并(多个有序来源)
4. 堆中存元组时注意比较规则
5. Python默认小根堆,大根堆存负数

**关键点:**
- 固定大小堆: 堆满时,新元素比堆顶大则替换
- 多路归并: 每个有序来源取当前最小,弹出一个后推入下一个
- 数据流中位数: 左大根堆(较小一半),右小根堆(较大一半)
- 堆中元组: (value, index)避免对象不能比较
- 时间复杂度: 取最值O(1),插入删除O(log n)

---

## 七、栈

**核心步骤:**
1. 普通栈: 处理嵌套结构、匹配
2. 单调栈: 找第一个更大/更小
3. 栈存值还是存下标
4. 确定单调方向
5. 处理边界情况

**关键点:**
- 括号匹配: 左括号入栈,右括号配对出栈
- 单调递减栈: 找下一个更大元素
- 单调递增栈: 找下一个更小元素
- 存下标: 需要算距离或位置关系
- 哨兵: 柱状图问题补0简化边界

---

## 八、贪心

**核心步骤:**
1. 确定每步局部最优策略
2. 证明局部最优能推出全局最优
3. 先排序再贪心
4. 维护当前最优状态
5. 更新状态和答案

**关键点:**
- 区间覆盖: 优先选择结束最早的
- 跳跃问题: 维护最远可达位置
- 买卖股票: 维护历史最低价格
- 最大子数组和: 前缀和为负则重新开始
- 前缀取舍: 有害前缀直接丢弃

---

## 九、链表

**核心步骤:**
1. 使用dummy节点统一处理
2. 反转链表(prev/cur/nxt三指针)
3. 快慢指针(找中点/环)
4. 合并链表
5. 修改指针前先保存下一跳

**关键点:**
- dummy节点: 头节点可能变化时使用
- 反转链表: prev=None, cur=head, nxt=cur.next, cur.next=prev
- 快慢指针: slow走一步,fast走两步
- 删除倒数第n个: fast先走n步,然后一起走
- 相交链表: 两个指针分别从两条链表头出发,走到末尾后跳到另一条链表头
- 两两交换: prev->second->first->next_pair

---

## 十、通用解题思路

**1. 识别题型:**
- 有序数组/答案空间单调 → 二分
- 枚举所有方案/组合排列 → 回溯
- 最大值/最小值/方案数/是否可行 → 动态规划
- 反复取最值/前k个 → 堆
- 嵌套结构/匹配/最近更大更小 → 栈
- 每步局部最优 → 贪心
- 树结构 → 递归/遍历
- 链表操作 → dummy/双指针/反转

**2. 固定思考顺序:**
- 递归函数的参数是什么?
- 结束条件是什么?
- 每一层从哪里开始枚举?
- 做选择时修改了哪些状态?
- 返回上一层时要撤销哪些状态?

**3. 易错点检查:**
- 边界条件是否完整?
- 初始化是否正确?
- 遍历顺序是否正确?
- 是否忘记撤销选择?
- 是否需要保存中间状态?
- 指针修改前是否保存下一跳?
