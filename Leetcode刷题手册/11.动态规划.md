# 动态规划

## 1. 核心思想

动态规划（Dynamic Programming，简称 DP）适合处理这类问题：

- 一个大问题可以拆成若干个更小的子问题
- 子问题会重复出现
- 每个子问题的答案可以被后面的状态复用
- 题目通常要求“最大值 / 最小值 / 方案数 / 是否可行”

它的本质可以概括成一句话：

> 先定义状态，再写状态转移。

很多题表面看起来不一样，但只要你发现：

- 当前答案依赖前面的答案
- 不想重复计算

就应该想到动态规划。

常见场景包括：

- 斐波那契型递推
- 从前往后做选择的线性问题
- 背包问题
- 网格路径问题
- 字符串匹配问题
- 区间合并或回文问题
- 多维状态转移问题

***

## 2. 动态规划五步法

写动态规划时，建议固定按下面顺序思考。

### 2.1 定义 `dp`

先明确 `dp` 数组的含义。

例如：

- `dp[i]`：表示到下标 `i` 为止的最优解
- `dp[i][j]`：表示在状态 `i, j` 下的答案
- `dp[i][j][k]`：表示三个变量共同决定时的答案

**这一步最重要。**

很多题不是不会写转移，而是一开始就没把 `dp` 定义清楚。

### 2.2 找状态转移方程

思考“当前状态由哪些更小状态转移而来”。

例如：

- `dp[i] = dp[i - 1] + dp[i - 2]`
- `dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])`
- `dp[i][j] = dp[i - 1][j] + dp[i][j - 1]`

### 2.3 确定初始化

初始化就是最小子问题的答案。

例如：

- `dp[0] = 1`
- `dp[0][0] = grid[0][0]`
- 第一行、第一列往往需要单独初始化

初始化错了，后面全错。

### 2.4 确定遍历顺序

遍历顺序的原则是：

- 当你算 `dp[i]` 时，它依赖的状态必须已经算出来

所以常见写法有：

- 从左到右
- 从上到下
- 按区间长度枚举
- 倒序遍历背包容量

### 2.5 确定返回值

最后想清楚答案到底是：

- `dp[n - 1]`
- `dp[m - 1][n - 1]`
- 整个 `dp` 数组中的最大值
- 某个特定边界位置

## 3. 基础模板

### 3.1 一维动态规划模板

```python
def dp_template(nums):
    n = len(nums)
    dp = [0] * n

    dp[0] = base_case

    for i in range(1, n):
        dp[i] = 由更小状态转移而来

    return dp[n - 1]
```

适用场景：

- 状态只和一个位置有关
- 当前答案依赖前面若干项
- 例如爬楼梯、打家劫舍、最长递增子序列

### 3.2 二维动态规划模板

```python
def dp_2d_template(m, n):
    dp = [[0] * n for _ in range(m)]

    初始化第一行和第一列

    for i in range(m):
        for j in range(n):
            dp[i][j] = 由上方、左方、左上方等状态转移

    return dp[m - 1][n - 1]
```

适用场景：

- 网格路径问题
- 两个字符串之间的匹配问题
- 区间或子串问题

### 3.3 记忆化搜索模板

有些题从“递归”更容易想，之后再配合缓存避免重复计算。

```python
from functools import lru_cache


def solve(nums):
    @lru_cache(None)
    def dfs(i):
        if i < 0:
            return base_case

        return 由更小子问题转移而来

    return dfs(len(nums) - 1)
```

它和动态规划本质一样，只是：

- 记忆化搜索：自顶向下
- 迭代 DP：自底向上

## 4. 写动态规划时最容易错的点

### 4.1 `dp` 的定义不清

例如 `dp[i]` 到底表示：

- 以 `i` 结尾的最优解
- 前 `i` 个元素的最优解

这两种定义完全不同，转移方程也不同。

### 4.2 初始化不完整

尤其是二维 DP，经常漏掉：

- 第一行
- 第一列
- 空字符串
- 容量为 `0` 的情况

### 4.3 遍历顺序写反

例如背包问题里：

- 0/1 背包通常容量要倒序遍历
- 完全背包通常容量要正序遍历

如果顺序错了，状态就会被错误复用。

### 4.4 只会背公式，不会从定义推出转移

动态规划最重要的不是记题，而是：

- 明确状态含义
- 明确“最后一步”发生了什么

只要最后一步能拆出来，通常就能写出转移。

### 4.5 过早做空间优化

很多题本来二维写法很清楚，但一开始就急着压缩成一维，反而容易写错。

建议顺序是：

1. 先写对
2. 再考虑优化空间

## 5. 动态规划常见题型

### 5.1 线性 DP

特点：

- 状态沿着数组或序列线性推进
- `dp[i]` 通常只依赖前几个位置

典型题：

- `70. 爬楼梯`
- `198. 打家劫舍`
- `152. 乘积最大子数组`

### 5.2 背包 DP

特点：

- 每个物品有“选 / 不选”或“选几次”的决策
- 常问是否能凑出、最多价值、最少数量

典型题：

- `416. 分割等和子集`（0/1 背包）
- `322. 零钱兑换`（完全背包）
- `279. 完全平方数`（完全背包）

### 5.3 前缀 / 切分型 DP

特点：

- `dp[i]` 往往表示前 `i` 个字符或前 `i` 个元素是否可行 / 最优
- 常见于字符串切分问题

典型题：

- `139. 单词拆分`
- `32. 最长有效括号`

### 5.4 二维路径 DP

特点：

- 状态在网格中移动
- 常由上方、左方转移而来

典型题：

- `62. 不同路径`
- `64. 最小路径和`
- `221. 最大正方形`

### 5.5 字符串双序列 DP

特点：

- `dp[i][j]` 表示两个字符串前缀之间的关系
- 常用于匹配、编辑、公共子序列

典型题：

- `1143. 最长公共子序列`
- `72. 编辑距离`

### 5.6 区间 DP

特点：

- 状态表示一个区间或子串
- 通常按区间长度递推

典型题：

- `5. 最长回文子串`

### 5.7 多维 DP

多维动态规划并不神秘，本质只是：

- 一个变量不够描述状态
- 需要用两个或更多维度同时刻画

例如：

- `dp[i][j]`：前 `i` 个字符与前 `j` 个字符的关系
- `dp[i][j]`：走到网格 `(i, j)` 的最优解
- `dp[i][j][k]`：位置 + 次数 + 状态 三者共同决定答案

所以“多维动态规划”只是状态维度更多，思路并没有变。

## 6. Hot 100 经典例题

下面按题型整理 Hot 100 里最经典的动态规划题。

## 6.1 70. 爬楼梯

题意：每次爬 `1` 或 `2` 阶，问爬到第 `n` 阶有多少种方法。

### 思路

到第 `i` 阶，最后一步只可能来自：

- 第 `i - 1` 阶走 `1` 步
- 第 `i - 2` 阶走 `2` 步

所以：

`dp[i] = dp[i - 1] + dp[i - 2]`

### 代码

```python
def climbStairs(n):
    if n <= 2:
        return n

    dp = [0] * (n + 1)
    dp[1] = 1
    dp[2] = 2

    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]

    return dp[n]
```

### 这题训练的能力

- 理解最基础的一维递推
- 学会从“最后一步”反推状态转移

***

## 6.2 198. 打家劫舍

题意：相邻房屋不能同时偷，问最多能偷多少钱。

### 思路

定义 `dp[i]` 表示偷到下标 `i` 为止的最大金额。

对于第 `i` 间房，有两种选择：

- 不偷它：`dp[i - 1]`
- 偷它：`dp[i - 2] + nums[i]`

所以：

`dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])`

### 代码

```python
def rob(nums):
    n = len(nums)
    if n == 1:
        return nums[0]

    dp = [0] * n
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])

    for i in range(2, n):
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])

    return dp[-1]
```

### 这题训练的能力

- 理解“选 / 不选”的线性 DP
- 体会 `dp[i]` 是“前 `i` 段范围的最优解”

***

## 6.3 322. 零钱兑换

题意：给定若干硬币面额，每种硬币可无限使用，求凑成目标金额的最少硬币数。

### 思路

定义 `dp[i]` 表示凑出金额 `i` 所需的最少硬币数。

如果当前选择硬币 `coin`，那么：

`dp[i] = min(dp[i], dp[i - coin] + 1)`

因为每种硬币可重复使用，所以这是完全背包。

### 代码

```python
def coinChange(coins, amount):
    INF = float("inf")
    dp = [INF] * (amount + 1)
    dp[0] = 0

    for coin in coins:
        for value in range(coin, amount + 1):
            dp[value] = min(dp[value], dp[value - coin] + 1)

    return dp[amount] if dp[amount] != INF else -1
```

### 这题训练的能力

- 认识完全背包
- 学会“最值型”背包的状态设计

***

## 6.4 416. 分割等和子集

题意：判断数组能否分成两个和相等的子集。

### 思路

设数组总和为 `total`。

如果能分成两部分相等，那么本质上就是：

- 能否从数组里选一些数，使它们的和恰好为 `total // 2`

这就是 0/1 背包。

定义 `dp[j]` 表示容量为 `j` 时是否可达。

### 代码

```python
def canPartition(nums):
    total = sum(nums)
    if total % 2 == 1:
        return False

    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True

    for num in nums:
        for j in range(target, num - 1, -1):
            dp[j] = dp[j] or dp[j - num]·

    return dp[target]
```

### 这题训练的能力

- 认识“是否可行型”背包
- 理解 0/1 背包为什么要倒序遍历

***

## 6.5 139. 单词拆分

题意：判断字符串 `s` 能否被字典中的单词完全拆分。

### 思路

定义 `dp[i]` 表示前 `i` 个字符是否可以被拆分。

如果存在某个 `j < i`，满足：

- `dp[j] == True`
- `s[j:i]` 在字典中

那么 `dp[i] = True`。

### 代码

```python
def wordBreak(s, wordDict):
    word_set = set(wordDict)
    n = len(s)
    dp = [False] * (n + 1)
    dp[0] = True

    for i in range(1, n + 1):
        for j in range(i):
            if dp[j] and s[j:i] in word_set:
                dp[i] = True
                break

    return dp[n]
```

### 这题训练的能力

- 理解“前缀是否可行”的 DP 写法
- 学会从切分点枚举状态转移

***

## 6.6 300. 最长递增子序列

题意：求数组中最长严格递增子序列的长度。

### 思路

定义 `dp[i]` 表示**以** **`nums[i]`** **结尾**的最长递增子序列长度。

如果 `nums[j] < nums[i]`，则可以把 `nums[i]` 接到 `j` 后面：

`dp[i] = max(dp[i], dp[j] + 1)`

### 代码

```python
def lengthOfLIS(nums):
    n = len(nums)
    dp = [1] * n

    for i in range(n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)
```

### 这题训练的能力

- 区分“前 `i` 个元素最优”与“以 `i` 结尾最优”
- 理解序列型 DP 的状态设计

## 6.7 62. 不同路径

题意：机器人从左上角走到右下角，每次只能向右或向下，问有多少种走法。

### 思路

定义 `dp[i][j]` 表示走到 `(i, j)` 的路径数。

由于当前位置只能从：

- 上方 `(i - 1, j)` 来
- 左方 `(i, j - 1)` 来

所以：

`dp[i][j] = dp[i - 1][j] + dp[i][j - 1]`

### 代码

```python
def uniquePaths(m, n):
    dp = [[1] * n for _ in range(m)]

    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]

    return dp[m - 1][n - 1]
```

### 这题训练的能力

- 理解二维路径型 DP
- 学会先初始化边界，再填内部状态

## 6.8 64. 最小路径和

题意：从左上角走到右下角，每次只能向右或向下，求路径数字和最小值。

### 思路

定义 `dp[i][j]` 表示走到 `(i, j)` 的最小路径和。

当前位置只能从上方或左方转移：

`dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]`

### 代码

```python
def minPathSum(grid):
    m, n = len(grid), len(grid[0])
    dp = [[0] * n for _ in range(m)]
    dp[0][0] = grid[0][0]

    for i in range(1, m):
        dp[i][0] = dp[i - 1][0] + grid[i][0]

    for j in range(1, n):
        dp[0][j] = dp[0][j - 1] + grid[0][j]

    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]

    return dp[m - 1][n - 1]
```

### 这题训练的能力

- 理解“计数型”和“最值型”二维 DP 的区别
- 练习二维数组初始化

## 6.9 5. 最长回文子串

题意：求字符串中最长的回文子串。

### 思路

定义 `dp[i][j]` 表示子串 `s[i:j + 1]` 是否是回文串。

转移条件：

- 如果 `s[i] != s[j]`，不是回文
- 如果 `s[i] == s[j]`，且中间部分也是回文，那么是回文

即：

`dp[i][j] = (s[i] == s[j]) and (j - i <= 2 or dp[i + 1][j - 1])`

这类题要按区间长度或从下往上枚举。

### 代码

```python
def longestPalindrome(s):
    n = len(s)
    dp = [[False] * n for _ in range(n)]
    start = 0
    max_len = 1

    for i in range(n - 1, -1, -1):
        for j in range(i, n):
            if s[i] == s[j] and (j - i <= 2 or dp[i + 1][j - 1]):
                dp[i][j] = True
                if j - i + 1 > max_len:
                    start = i
                    max_len = j - i + 1

    return s[start:start + max_len]
```

### 这题训练的能力

- 理解区间 DP
- 明白为什么有些题必须倒序遍历 `i`

## 6.10 1143. 最长公共子序列

题意：求两个字符串的最长公共子序列长度。

### 思路

定义 `dp[i][j]` 表示 `text1` 前 `i` 个字符和 `text2` 前 `j` 个字符的最长公共子序列长度。

如果最后一个字符相同：

- `dp[i][j] = dp[i - 1][j - 1] + 1`

如果不同：

- `dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])`

### 代码

```python
def longestCommonSubsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

### 这题训练的能力

- 掌握双序列二维 DP
- 理解“前缀 vs 前缀”的状态定义方式

## 6.11 72. 编辑距离

题意：求把 `word1` 变成 `word2` 所需的最少操作数。

操作包括：

- 插入一个字符
- 删除一个字符
- 替换一个字符

### 思路

定义 `dp[i][j]` 表示 `word1` 前 `i` 个字符变成 `word2` 前 `j` 个字符的最少操作数。

如果最后一个字符相同：

- `dp[i][j] = dp[i - 1][j - 1]`

如果不同：

- 插入：`dp[i][j - 1] + 1`
- 删除：`dp[i - 1][j] + 1`
- 替换：`dp[i - 1][j - 1] + 1`

取三者最小值。

### 代码

```python
def minDistance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,
                    dp[i][j - 1] + 1,
                    dp[i - 1][j - 1] + 1,
                )

    return dp[m][n]
```

### 这题训练的能力

- 理解多种操作同时参与转移
- 熟悉二维 DP 的边界初始化

## 8. 一句话总结动态规划

做动态规划题时，优先问自己四个问题：

- `dp` 数组的含义是什么？
- 当前状态由哪些更小状态转移而来？
- 初始状态怎么设？
- 遍历顺序为什么这样写？

只要这四个问题能答清楚，大多数动态规划题都能拆开。
