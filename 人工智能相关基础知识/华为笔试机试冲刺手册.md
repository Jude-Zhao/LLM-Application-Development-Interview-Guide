## 1. 先看结论

### 1.1 华为研发岗一定要准备机试

根据华为招聘官网 FAQ，研发类岗位在简历筛选通过后，需要参加上机考试和综合测评，之后才进入专业面试和主管面试阶段。也就是说，只要你投的是研发/算法/AI/软件开发相关岗位，就不要假设自己只会遇到纯面试。  
官方参考：<https://career.huawei.com/reccampportal/portal5/faq.html>

### 1.2 近一年至少有两种主流题型

#### 题型 A：通用研发/开发类机试

- 常见形态：120 分钟，3 道算法题
- 常见分值：100 / 200 / 300
- 常见通过线：150 分左右
- 常见平台与模式：牛客，ACM 输入输出，本地 IDE 可辅助编写

这类题型更像经典算法机考，重点在：

- 哈希、字符串、数组
- 双指针、滑窗、前缀和
- DFS / BFS
- 二分、贪心
- 树、堆、图最短路
- 动态规划

#### 题型 B：AI 岗机考

2025 年起，社区普遍反馈华为 AI 岗机考发生明显变化，不再只是 3 道算法题，而是：

- 选择题 + 编程题混合
- 选择题覆盖机器学习、深度学习、线性代数、概率统计、数值计算
- 编程题开始出现“AI 场景包装题”

从 2025-09-03 和 2025-09-12 两批社区真题看，AI 岗高概率会出现：

- 15 道单选 + 5 道多选
- 2 道编程题
- 编程题题面披着 AI/大模型外衣，但本质仍是算法、数据处理或基础 ML 实现

一句话总结：

**如果你投的是 AI/算法/机器学习相关岗位，今晚不要只刷普通算法题，必须补机器学习、概率统计和 Transformer 基础。**

## 2. 今晚的复习优先级

如果你时间有限，按下面顺序复习，收益最高。

### 优先级 1：算法做题能力

先保证你能拿到基础分和中档分。

必须熟练：

- 哈希表计数、去重、映射
- 数组排序 + 自定义排序
- 双指针 / 滑动窗口
- DFS / BFS / 回溯
- 二分答案
- 堆 / 优先队列
- 树的遍历、祖先、路径
- Dijkstra 最短路
- 动态规划：线性 DP、背包味 DP、区间/状态转移基础
- 逆序对 / 树状数组这种“会考但不一定每次考”的题型

### 优先级 2：机器学习与 AI 基础

必须会解释：

- 过拟合 / 欠拟合
- 逻辑回归
- L1 / L2 正则
- 决策树 / 随机森林 / GBDT
- KMeans / GMM / EM
- PCA
- 常见激活函数
- Transformer 编码器 / 解码器 / Attention
- MoE 基础概念
- 分类评估指标

### 优先级 3：概率统计与线性代数

重点记：

- 期望、方差、协方差
- 条件概率、贝叶斯公式
- 伯努利 / 二项 / 泊松 / 均匀 / 正态 / 指数分布
- 泊松近似二项分布
- 线性变换的性质
- 特征值 / 特征向量 / 方差贡献率
- 数值迭代、牛顿法的收敛阶

### 优先级 4：考试策略

- 先拿稳前两题，不要第三题开局硬刚到底
- AI 选择题不要靠“感觉”，很多题是概念边界题
- 读题时先把场景皮去掉，看本质是什么算法/模型

## 3. 高频知识点速记

### 3.1 机器学习

### 3.1.1 过拟合与欠拟合

- 训练集效果好、测试集效果差：过拟合
- 训练集和测试集都差：欠拟合
- 解决过拟合：
  - 增加数据
  - 正则化
  - 降低模型复杂度
  - 早停
  - Dropout

### 3.1.2 线性回归 vs 逻辑回归

- 线性回归：
  - 输出连续值
  - 常用损失：MSE
- 逻辑回归：
  - 输出分类概率
  - `p = sigmoid(wx + b)`
  - 二分类常用损失：交叉熵

你要会回答：

- 为什么逻辑回归虽然名字有“回归”，却常用来做分类
- 为什么逻辑回归输出的是概率
- 为什么分类任务常用交叉熵而不是 MSE

### 3.1.3 L1 与 L2 正则

- L1：
  - 倾向产生稀疏解
  - 可做特征选择
- L2：
  - 让参数整体变小但不一定变 0
  - 更平滑、更常见

面试/选择题高频点：

- “稀疏”优先想到 L1
- “防止权重过大、提升泛化”优先想到 L2

### 3.1.4 决策树 / 随机森林 / GBDT

- 决策树：
  - 单树，容易过拟合
- 随机森林：
  - Bagging
  - 多棵树并行训练
  - 通过随机采样和特征子集降低方差
- GBDT：
  - Boosting
  - 串行加树，后面的树拟合前面的残差
  - 偏差下降明显

简单记忆：

- 随机森林更偏“稳”
- GBDT 更偏“强”

### 3.1.5 KMeans / GMM / EM

- KMeans：
  - 硬聚类
  - 样本只属于一个簇
- GMM：
  - 软聚类
  - 样本属于各个簇有概率
- EM：
  - 常用于含隐变量模型
  - E 步：算隐变量后验/责任度
  - M 步：最大化期望对数似然更新参数

高频陷阱：

- EM 不是某一个模型专属，但 GMM 常用 EM 求解
- GMM 不要求“必须对角协方差”才能使用 EM

### 3.1.6 PCA

- 目标：降维、去冗余、压缩表示
- 本质：找方差最大的方向作为主成分
- 主成分选择依据：**方差贡献率**

高频错法：

- 不是按类别分布选主成分
- 不是按样本密度选主成分

### 3.1.7 分类指标

- Accuracy：整体正确率
- Precision：预测为正里有多少是真的正
- Recall：真实为正里找回了多少
- F1：Precision 和 Recall 的调和平均
- ROC-AUC：阈值变化下的整体排序能力

遇到类别不平衡时：

- 不要只看 Accuracy
- 更应看 Precision / Recall / F1 / AUC

---

### 3.2 深度学习与大模型

### 3.2.1 常见激活函数

- ReLU：`max(0, x)`
  - 负数输出 0，正数原样输出
- Sigmoid：
  - 输出范围 `(0, 1)`
  - 常用于二分类概率输出
- Tanh：
  - 输出范围 `(-1, 1)`
- Softmax：
  - 多分类输出概率，所有类别概率和为 1

必须记住：

- Sigmoid 在输入 0 附近梯度最大
- ReLU 简单高效，但可能有“死亡 ReLU”
- Softmax 是“归一化为概率分布”

### 3.2.2 Transformer

核心结构：

- 输入嵌入 + 位置编码
- 多头自注意力
- 前馈网络
- 残差连接 + LayerNorm

### 3.2.3 编码器与解码器

- 编码器：处理输入序列表示
- 解码器：
  - 有 Masked Self-Attention
  - 还会有 Cross-Attention
  - Cross-Attention 的 K、V 通常来自编码器输出

高频易错点：

- **带 Mask 的通常是解码器的自注意力层**
- 解码器里第二个注意力层常是 Encoder-Decoder Attention

### 3.2.4 Attention 一定要会

至少会这几件事：

- Q、K、V 是什么
- 点积注意力怎么计算
- 为什么要除以 `sqrt(d_k)`
- Mask 是为了什么
- 多头注意力的意义是什么

如果编程题考注意力，本质多半是：

- 矩阵乘法
- 维度变换
- softmax
- 掩码处理

### 3.2.5 MoE

MoE 常见考法不是让你推导前沿论文，而是问：

- token 如何路由到专家
- top-k 专家如何选
- 为什么要做负载均衡
- 为什么要限制路由到少量设备

一句话理解：

**MoE 的重点是“专家选择 + 通信成本 + 负载均衡”。**

### 3.3 概率论与统计

### 3.3.1 期望与方差

必须秒写：

- `E[aX + b] = aE[X] + b`
- `Var(aX + b) = a^2 Var(X)`

例如：

- 若 `Var(X) = 4`
- `Y = 3X - 5`
- 则 `Var(Y) = 9 * 4 = 36`

### 3.3.2 常见分布

- 伯努利分布：
  - 一次 0/1 试验
- 二项分布：
  - 多次独立伯努利试验成功次数
- 泊松分布：
  - 单位时间/区域内稀有事件计数
- 均匀分布：
  - 区间内“等可能”
- 正态分布：
  - 连续型经典钟形分布
- 指数分布：
  - 泊松过程的到达间隔

高频结论：

- 当 `n` 大、`p` 小、`np = λ` 有限时，二项分布可近似为泊松分布
- 泊松过程的首次到达时间服从指数分布

### 3.3.3 正态性检验

常见可用于判断正态性的思路：

- 看直方图/QQ 图
- 拟合优度检验
- 偏度和峰度

高频陷阱：

- **t 检验不是“正态性检验方法”本身**

### 3.3.4 贝叶斯

必须会：

- 条件概率定义
- 全概率公式
- 贝叶斯公式

很多题的本质不是公式难，而是事件定义混乱。做题时先写清：

- 已知什么
- 要求什么
- 分母是“证据”

---

### 3.4 线性代数与数值计算

### 3.4.1 线性变换

线性变换最核心的两个性质：

- `T(u + v) = T(u) + T(v)`
- `T(cu) = cT(u)`

直接判题技巧：

- 有常数项，通常就不是线性变换
- 零向量必须映到零向量

### 3.4.2 外积

如果 `A = uv^T`，那么：

- `A[i][j] = u[i] * v[j]`

这是选择题里很喜欢出的秒杀点。

### 3.4.3 特征值 / 特征向量

PCA、矩阵对角化、稳定性分析都可能挂钩。  
临考前至少保证你知道：

- 特征向量方向不变
- 特征值表示该方向上的缩放倍数
- PCA 取协方差矩阵的主特征方向

### 3.4.4 数值迭代与牛顿法

必须区分：

- 不动点迭代
- 牛顿迭代

常见结论：

- 牛顿法对单根通常有二次收敛
- 对重根收敛速度会下降
- 不是所有初值都会收敛到同一个根

### 3.5 算法与数据结构

### 3.5.1 哈希

高频用途：

- 计数
- 去重
- 频次统计
- 前缀和配对
- 快速判断是否出现过

### 3.5.2 双指针 / 滑窗

适用信号：

- 连续子数组/子串
- 最长/最短满足条件区间
- 去重
- 固定窗口或可伸缩窗口

### 3.5.3 DFS / BFS

DFS 常见：

- 连通块
- 树上搜索
- 回溯
- 祖先关系

BFS 常见：

- 最短步数
- 层序遍历
- 状态扩展

### 3.5.4 二分

不仅是“有序数组找值”，还要会：

- 二分答案
- 判定函数单调性

### 3.5.5 Dijkstra

一看到：

- 带权图
- 边权非负
- 最短路

优先考虑 Dijkstra。

### 3.5.6 动态规划

临考前最值得稳住的 DP 模型：

- 爬楼梯/打家劫舍式线性 DP
- 背包型 DP
- 子序列 DP
- 区间 DP 的基本认知

不会写复杂 DP 时，至少先问自己：

- 状态是什么
- 转移从哪里来
- 初始条件是什么
- 顺序怎么遍历

### 3.5.7 树状数组 / 逆序对

不是每次都考，但华为真题确实出现过。  
用途：

- 动态前缀和
- 逆序对统计
- 局部更新 + 前缀查询

## 5. 你明天最可能碰到的“坑”

### 5.1 只刷 LeetCode 热题，不看 AI 基础

如果你投的是 AI/算法岗，这会非常危险。  
你可能会在选择题里直接丢一大截分。

### 5.2 会概念，不会算

比如：

- 方差线性变换
- 泊松近似
- 线性变换是否成立
- Attention 的维度与计算

这些题不是背定义就够，得能快速算。

### 5.3 读不懂 ACM 输入输出

很多人不是不会算法，而是：

- 读入格式没处理好
- 多组输入漏写
- 输出顺序错
- 空格/换行错

### 5.4 被“AI 场景外衣”吓住

题目写成：

- 设备故障预测
- 大模型显存优化
- MoE 路由
- 注意力计算

你要做的是先剥皮，看它到底是：

- 数据清洗
- 矩阵计算
- top-k
- 分组
- DFS / BFS
- 哈希
- 排序

## 7. 考场策略

### 7.1 做题顺序

推荐顺序：

1. 先快速浏览全卷
2. 先拿稳最容易得分的选择题或前两道编程题
3. 对高分难题先做“可得部分”
4. 最后再回头优化

### 7.2 选择题策略

- 先排除明显错项
- 对概念边界题，优先想“定义是否严格成立”
- 对数学题，别心算过度，纸上列式反而更稳

### 7.3 编程题策略

每道题固定做这 5 步：

1. 把场景翻译成算法语言
2. 明确输入输出
3. 写最朴素可运行版本
4. 用样例手推一遍
5. 再考虑优化

### 7.4 语言选择

- 只用你最熟的语言
- 如果 Python 最熟，就用 Python
- 如果你平时 C++ 更稳，不要临时换 Python

### 7.5 容易失误的细节

- 忘记提交
- 只本地跑过，没按平台格式测试
- 样例过了就以为没问题
- 忽略边界：
  - 空数组
  - 单节点
  - 重复值
  - 负数
  - 多组输入

### 7.6 远程机考环境检查

根据近年机考指南与经验贴，远程机考常见会要求浏览器监控、摄像头和手机防作弊配合。明天开考前至少检查：

- Chrome 浏览器是否可正常进入考试页面
- 电脑摄像头、麦克风是否可用
- 手机是否有电，并能保持常亮
- 微信、QQ、Outlook、飞书等弹窗软件是否已关闭
- 本地 IDE 是否已提前打开并测试
- 网络是否稳定

如果考试页面明确要求手机扫码监控，就不要在考试中途切出监控页面，以免被误判。

## 8. 高频概念最后速背

把下面这些背熟，明天很值钱。

- 训练高、测试低：过拟合
- L1 稀疏，L2 平滑
- PCA 按方差贡献率选主成分
- GMM 常用 EM 求解
- 泊松过程首次到达时间服从指数分布
- `Var(aX+b)=a^2 Var(X)`
- 有常数项通常不是线性变换
- 解码器有 Masked Self-Attention
- 解码器 Cross-Attention 的 K、V 来自编码器输出
- Softmax 输出和为 1
- 二项分布在 `n` 大 `p` 小时可用泊松近似
- 非负权最短路优先想 Dijkstra
- 连续区间题优先想滑窗 / 前缀和
- 树上祖先问题优先想 DFS 序 / 父节点记录 / LCA 思维
